宅急送速运BOS V2.0系统

发表于 2017-11-27 | 分类于 项目

author : 李旭岚

项目背景

　　速运 BOS 物流系统是宅急送公司的核心综合业务系统，实现“决策、管理、操作”一体化运作。BOS v2.0 主要是在原有陈旧的技术架构基础上，建立起一套完整的前端系统、服务器端系统、外围系统，打造全方位还原，一流的物流平台环境。

项目主要技术点

  • 01-项目采用maven管理，通过模块集成，复用jar包
  • 02-项目采用Git作为版本控制，替换SVN ，同时讲解使用“码云”管理git项目代码
    码云平台帮助文档
  • 03-项目分为 bos_fore前端系统、bos_management 后台系统、crm_management 客户管理系统、bos_sms短信平台、bos_mail邮件平台、 activeMQ 消息队列、redis 内存数据库、elasticSearch 搜索平台、多系统交互。
  • 04-项目服务器采用 struts2+spring4+spring data jpa+ jpa（hibernate实现）完成数据层架构，采用spring data 实现
  • 05-后台系统页面 采用jQuery Easyui + Ztree实现
  • 06-前台系统页面 采用BootStrap + AngularJS 实现 ，使用city-picker插件，实现省市区三级联动
  • 07-系统之间通信采用 Apache CXF 基于 restful风格 服务 (webservice技术)
  • 08-第三方短信平台 吉信通 发送短信验证码
  • 09-spring data redis 使用 、redis内存数据库、javamail发送邮件
  • 10-ActiveMQ 消息队列使用
  • 11-KindEditor 在线富文本编辑器
  • 12-Bootstrap + Angular 自定义分页实现
  • 13-freemaker页面静态化技术、Quartz 任务调度定时框架 (quartz+spring)
  • 14-ElasticSearch + ik分词器 spring data elasticsearch搜索平台使用
  • 15-第三方 百度地图API（地址自动补全、lbs云存储、动态路径规划）
  • 16-采用Apache Shiro 进行权限控制 （shiro登录、第三方登录、SSO单点登录）
  • 17-ehcache缓存权限数据和业务数据
  • 18-采用POI、Itext、JasperReport+ireport等Java技术生成报表(参考easypoi框架)
  • 19-采用highchart等报表展示
  • 20-POI数据上传下载（主要是Excel表格导入、导出）
  • 21-将项目部署在Linux系统上（涉及：虚拟机安装、安装Linux系统、安装配置JDK、安装配置Oracle、安装配置Tomcat、 安装配置Redis、安装配置ActiveMQ、安装配置ElasticSearch等）

项目主要功能点

一、基础数据模块

收派标准功能开发
快递员管理功能(列表、添加、作废功能)
区域数据导入功能
定区管理功能
定区添加、定区带条件分页列表查询、定区关联客户、定区关联快递员
分区管理功能

二、宣传活动任务管理(cms)

宣传活动后台功能
后台添加宣传活动
宣传活动列表展示
宣传活动前台功能
前台分页列表展示功能
宣传活动详情页展示
宣传活动自动过期功能

三、订单、工单、运单：

前台系统的注册、登录功能、邮箱绑定功能(邮箱激活码)
客户登录前台系统(bos_fore)，在线下订单功能
后台系统(bos_management)根据订单生成工单(复杂业务：订单的自动分单功能)
同时给负责的快递员发送短信(取派任务)
运单快速录入功能
运单录入功能
依据订单数据进行运单录入功能
依据运单快速录入的数据进行运单的录入功能
添加运单时同步创建运单数据的索引
运单管理功能：
运单的分页条件查询功能
运单数据同步索引库功能
根据运单数据生成报表，进行报表的导入导出功能

四、使用shiro对后台系统(bos_management)进行权限控制

粗粒度和细粒度的权限控制
授权数据管理
用户数据、权限数据、角色数据、菜单数据(rbac)
动态菜单管理
用户注销功能
缓存权限数据、缓存业务数据

五、运输配送管理

运单管理中开启中转配送(生成运输配送信息)
运输路径的百度地图显示
运输配送信息列表展示功能
出入库操作功能
开始配送功能
签收录入功能

六、系统报表导出功能

Excel报表
pdf报表
前端页面报表展示
